# Project Space Selection

## Goal

Enable autodiscovered project navigation from `~/projects`, including grouped projects, explicit workspace selection, worktree selection, and opening the current target in a selected desktop app.

## Checklist

- [x] Persist local UI state under `~/.project-space`
- [x] Autodiscover projects from hardcoded `~/projects`
- [x] Support standalone projects
- [x] Support workspace-style projects with `base/` and worktrees
- [x] Support project groups with drill-down navigation
- [x] Keep root/group navigation in the sidebar spaces area
- [x] Add explicit `Workspace` selection in the explorer
- [x] Keep worktree selection in the explorer for workspace projects
- [x] Show the selected workspace or worktree path in the main view
- [x] Add a file explorer view for the selected target
- [x] Add an open-in-app dropdown in the main header
- [x] Load launcher choices from a hardcoded registry
- [x] Show only installed launcher apps
